# Support

Verdict è un progetto solo/hobby, non c'è un servizio clienti. Ma ci sono canali
ordinati per il tipo di richiesta:

## Serve aiuto per usare Verdict?

1. Leggi [docs/FAQ.md](docs/FAQ.md) — copre le domande frequenti.
2. Cerca nelle [Issues chiuse](https://github.com/Leongithacc/Verdict/issues?q=is%3Aissue+is%3Aclosed).
3. Se non trovi risposta, apri una [nuova Issue](https://github.com/Leongithacc/Verdict/issues/new/choose)
   con il template "Bug report" o "Feature request".

## Hai trovato un bug?

Apri una Issue con il template Bug report. **Includi**:
- Versione Verdict (`wpep version`)
- Sistema (Windows 10/11 build)
- Cosa hai fatto, cosa ti aspettavi, cosa è successo
- Log se disponibili

## Vuoi proporre una feature?

Apri una Issue con il template Feature request. Per un nuovo tweak KB, ricorda
la **regola d'oro**: serve una fonte primaria vendor (Microsoft / NVIDIA / AMD /
Intel / Riot / ecc.). Senza fonte, il tweak non viene accettato — è una
promessa di credibilità che facciamo agli utenti.

## Hai un problema di sicurezza?

**Non** aprire una Issue pubblica. Vedi [SECURITY.md](SECURITY.md) per il
processo di responsible disclosure.

## Discussione generale / brainstorming

Se abilitate, le [Discussions](https://github.com/Leongithacc/Verdict/discussions)
sono il posto giusto per domande aperte, proposte non ancora feature request,
showcase di configurazioni proprie.

## Vuoi contribuire codice?

Leggi [CONTRIBUTING.md](CONTRIBUTING.md). Il flow è: fork → branch → PR con test
verdi → review.

## Community backend (verdict-community)

Se il tuo problema riguarda il Worker Cloudflare (endpoint /v1/*) e non l'app
Windows, vai su:
[github.com/Leongithacc/verdict-community/issues](https://github.com/Leongithacc/verdict-community/issues).

## Tempi risposta realistici

Progetto solo/hobby, nessun SLA. Tipicamente:
- Bug critici / sicurezza: entro 1 settimana
- Feature request: valutata all'occasione, spesso in un ciclo di release
- Domande di uso: quando riesco, ma la FAQ copre la maggior parte

Preferisco rispondere una volta bene che 100 volte in fretta.
