# Code of Conduct

Verdict è un progetto solo/hobby con community piccola. Le regole sono corte:

## Sii diretto

- Rispondi al **contenuto**, non a chi lo scrive.
- Le "domande stupide" non esistono: se il progetto non le rende ovvie,
  è un bug del progetto, non del chi chiede.
- Il feedback tecnico duro è benvenuto. Gli attacchi personali no.

## Rispetta la regola d'oro

Questo NON è un forum di tweak. Ogni proposta di tweak richiede una **fonte
primaria verificata** (vendor: Microsoft / NVIDIA / AMD / Intel / Riot / ...).
Vedi [CONTRIBUTING.md](CONTRIBUTING.md).

- "Un YouTuber ha detto" non è una fonte.
- "Il mio amico X ha guadagnato Y FPS" non è una misura.
- "Su Reddit tutti fanno" non è un argomento.

Chi ignora la regola d'oro dopo un warning viene ignorato — lo scopo del
progetto è la credibilità onesta, non il traffico.

## Non tollerato

- Molestie, insulti personali, discriminazione (razziale, di genere, orientamento, ecc.)
- Spam / marketing di prodotti competitor senza dichiararlo
- Doxxing o pubblicazione di dati privati altrui
- Retorica "gaming optimizer +30 FPS garantiti" — quel tono di marketing è
  esattamente cosa Verdict rifiuta

## Enforcement

Per report:
- Segnala in una issue con `label: coc-report` (o via email nel profilo GitHub
  se preferisci privato).
- L'autore ([@Leongithacc](https://github.com/Leongithacc)) valuta caso per
  caso. Sanzioni possibili: warning, ban temporaneo, ban permanente dai canali
  progetto (issue, PR, discussion).

## Fonti

Questo CoC è ispirato dai principi generali del
[Contributor Covenant](https://www.contributor-covenant.org/), ridotto alle
tre regole che contano per un progetto piccolo di community tecnica.
